# Game UI Wireframe Patterns

Quick reference for game interface design. Covers menus, HUDs, and in-game UI elements.

---

## Main Menu

### Structure

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│                                                                             │
│                           [GAME LOGO]                                       │
│                                                                             │
│                                                                             │
│                        ┌─────────────────┐                                  │
│                        │    NEW GAME     │                                  │
│                        └─────────────────┘                                  │
│                        ┌─────────────────┐                                  │
│                        │    CONTINUE     │                                  │
│                        └─────────────────┘                                  │
│                        ┌─────────────────┐                                  │
│                        │    OPTIONS      │                                  │
│                        └─────────────────┘                                  │
│                        ┌─────────────────┐                                  │
│                        │    CREDITS      │                                  │
│                        └─────────────────┘                                  │
│                        ┌─────────────────┐                                  │
│                        │     EXIT        │                                  │
│                        └─────────────────┘                                  │
│                                                                             │
│  v1.0.3                                          © 2024 Studio Name        │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Key Principles

- Logo prominent at top (30-40% of screen height)
- Menu items centered, stacked vertically
- Clear hover/selected states (glow, scale, color shift)
- Version number bottom-left
- Copyright bottom-right
- Background: animated scene, artwork, or subtle particles

### Menu Button States

```
Default:     ┌─────────────────┐
             │    NEW GAME     │  ← Normal color
             └─────────────────┘

Hover:       ╔═════════════════╗
             ║    NEW GAME     ║  ← Brighter, glow effect
             ╚═════════════════╝

Selected:    ▶ ════════════════ ◀
               ║  NEW GAME  ║      ← Indicator arrows, highlight
             ▶ ════════════════ ◀

Disabled:    ┌─────────────────┐
             │    CONTINUE     │  ← Grayed out (50% opacity)
             └─────────────────┘
```

```
│   ░ = blurred/dimmed game background                                        │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Key Principles

- Game visible but dimmed/blurred behind (50-70% opacity overlay)
- Centered panel with clear boundary
- "RESUME" as first option (most common action)
- ESC or same button to resume
- Confirm before "Quit to Main Menu"

### Transition Manager Pattern

```
SCREEN STACK:
────────────────────────────────────────────────────────────────────────────

Stack: [ MainMenu, Options, Graphics ]
                              ▲
                              └── Top = active screen

Operations:
────────────────────────────────────────────────────────────────────────────
Push(Screen)   → Add to top, animate in, disable previous
Pop()          → Remove top, animate out, enable previous
Replace(Screen)→ Remove top, add new (no stack change)
Clear()        → Remove all, return to root

Animation Callbacks:
────────────────────────────────────────────────────────────────────────────
OnTransitionStart()    // Lock input during animation
OnTransitionComplete() // Unlock input, enable new screen
OnScreenEnabled()      // Screen becomes active
OnScreenDisabled()     // Screen becomes inactive (but may remain visible)
```

## Best Practices

### Game Menu UX

1. **First option = most common action** (Resume, Start, Continue)
2. **Consistent navigation** across all menus (B/ESC = Back)
3. **Remember last selection** when returning to menu
4. **Preview changes** before applying (graphics, audio)
5. **Warn before destructive actions** (delete save, quit without saving)

### In-Game HUD

1. **Minimize clutter** - show only essential info
2. **Consistent positions** - health always same spot
3. **Scalable UI** for different resolutions
4. **Hide when not needed** (auto-hide HUD option)
5. **Colorblind considerations** - don't rely on color alone

### State Machine Guidelines

1. **Single active state** - Only one menu state active at a time
2. **Explicit transitions** - Define all valid state changes
3. **Enter/Exit hooks** - Clean up resources on state exit
4. **Input gating** - Disable gameplay input when menu is open
5. **Back stack** - Maintain history for intuitive back navigation

### Focus Navigation Guidelines

1. **Always have something focused** - Default focus on menu open
2. **Visual focus indicator** - Clear highlight on focused element
3. **Wrap navigation** - Optional wrap at list boundaries
4. **Sticky column** - Remember column when navigating grid rows
5. **Sound feedback** - Audio cue on focus change

### Accessibility

- [ ] Remappable controls
- [ ] Subtitles with size options
- [ ] Colorblind mode
- [ ] UI scale options
- [ ] Hold vs toggle options
- [ ] Audio cues for visual elements
- [ ] Keyboard/gamepad navigation for all UI
- [ ] Screen reader support for menus
